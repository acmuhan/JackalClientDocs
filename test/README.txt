  基本操作
  
  · 打开客户端后，如果你启用了 Console 模块，就会出现一个黑窗口，该窗口称作控制台，输出一些关键信息。可以关闭 Console 模块将其关闭。命令行模块 Shell 可以输入一些命令 (COMMANDS.TXT 中有详细教程)。
  · 按下 右Ctrl/右Shift 打开/关闭菜单（也叫 ClickGUI）。这个快捷键可以在 GUI 选项卡中的 Toggle Key 中修改。
  · 打开菜单后，你能看到很多模块，鼠标放在上面可以显示简介。左键切换开关，右键进入模块配置。每一列模块顶部是列标题，单击可以折叠/展开，也可以进行拖动。按箭头键可以移动视角，这在模块出屏幕的时候会很有用。按 Ctrl + 滚轮可以调节缩放。
  · 本客户端在打开后就是置顶状态，如果被最小化，可以按 右Ctrl/右Shift 还原。如果效果不明显，可考虑启用 ForceTopmost 强制置顶。
  · 打开菜单后按 Ctrl+F 可以进入模块搜索界面。会显示所有可用模块。
  · 关闭控制台或者单击 Exit 模块退出客户端（有一个默认的快捷键：RAlt+Delete）。 Restart 用于重新启动。
  · 如果实在不知道怎么退出，先看任务栏确保当前窗口是客户端，然后按 Alt+F4 关闭窗口。

  使用须知
  
 1. 客户端程序为 Release 文件夹下的 JackalClient.exe
 2. 请注意客户端的更新，每一次更新都会发在群里，B站视频下方的链接可能不会更新。值得注意的是，运行启动器 Launcher.exe 会自动检查更新，如果有新版本就会自动提示下载。如果更新失败，请改天再试（因为蓝奏云解析不稳定）。
 3. 本客户端可能存在一些不安全的模块。在 Config 选项卡下，Block Unsafe Modules 选项开启后，将会阻止这些模块的启用。请务必不要使用这些模块做违法背德的事。
 4. 本客户端存在一些利用驱动程序实现的模块。这些模块目前不开放。
 5. 本客户端存在一些会被识别为恶意程序 (Malware) 的模块。使用这些模块时，请：
 
	① 非常明白这个模块是干嘛的。
	② 如果不放心，务必在虚拟环境中测试。
	③ 仅作学习或实验用途，不做危害他人、违法的事。
	④ 如果出事，请自行承担责任，工具作者概不负责。
	
 6. 如果杀软对本客户端进行报毒（例如火绒的 Hetile.e/Exploit，360的 Uacbypass 等），请考虑添加信任或者加入排除项。
 7. 如遇客户端无法启动（启动就崩溃），可考虑删除配置文件，具体位置在config下的 profile-数字.json 文件。一般崩溃都是因为配置出了问题。可以先把这个文件移到别的位置，运行客户端看看情况是否好转。如果还是崩溃，再把 json 放回来，对其他文件进行同样的操作。注意：不要对 default.json 进行操作，那是默认配置。如果实在不行，请反馈给作者，并将配置呈现给作者。
 8. 本客户端对 Win10 以下计算机兼容性较差，可能会出现打开直接报错（一些函数找不到）的情况。如遇这种情况，可以先向作者反馈。尽量使用 Win10 及以上版本的 Windows 系统。
 9. 本客户端是32位程序，但是对32位系统兼容性不是很好。有些注入类模块无法成功在32位系统上运行。附属程序 loader等都是64位的程序，无法在32位系统上运行。
 10. 作者对客户端进行了汉化，以防一些用户看不懂。在 GUI 中的 Language/语言 选择简体中文 即可。作者对大部分英文都进行了汉化。
 11. 本客户端基于 raylib图形库/OpenGL。 没有手机版。没有 Linux 版。没有 Mac 版。
 12. 某些 NVIDIA 或者 AMD 显卡驱动有bug, 会让客户端透明位置变成黑色。请换用集显或者其他独显。你可以填写群里的黑屏问题收集表。还有，有时候黑屏是因为开了一些美化软件，关闭后看看是否恢复正常。详细问题请见仓库 https://github.com/noexcept2005/JackalClient/issues/3
 13. 如果软件是测试版本 (Debug Version)，会有更多限制。
 14. 如果你的计算机在运行本客户端时出现卡顿，请考虑关闭着色器模块（Shader），将 HUD 中的Modules Color调为单色，关闭 ForceTopmost, Vape 等一些高消耗模块，在 Config 选项卡中关闭 Always Run.
 15. 如果本软件目录中有缺失的内容，可以在客户端中启用 Self Restore 模块进行还原。原理就是将 backup 文件夹下的加密的文件解密还原出来。 Self Backup 则是逆操作，用于备份。 Auto Restore 模块会自动检查，如果缺失则立即还原。
 16. 本客户端会将通知等内容记录到日志 CLIENT.LOG 中，你可以进行查看。每次启动客户端会重置其内容，注意。如果崩溃，请向作者提供 crashlog下的 last_crash.log
 17. 本客户端不是 Minecraft 黑客端，只是借鉴其视觉。
 18. 请不要对本工具进行逆向破解。混淆和加壳会损失很多兼容性，对用户造成不便。
 19. 本客户端由于赶工匆忙，可能存在大量的漏洞和瑕疵。如果你有好的建议，欢迎反馈。作者会尽量修的。
