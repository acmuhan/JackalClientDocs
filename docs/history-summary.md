# 版本与演进

## 历史节奏

根据 `test/HISTORY.TXT`，项目更新频繁，呈现以下特征：

- 高频迭代（修复 + 新增并行）
- GUI 体验持续增强
- 命令系统不断扩展
- 在线能力持续补全

## 重点演进方向

1. 命令能力扩展
命令体系从基础控制逐步覆盖文件、系统、网络、在线服务、自动化流程。

2. UI/UX 精细化
大量版本围绕动画、布局、显示策略、交互细节持续迭代。

3. 模块化深化
模块数量和配置项显著增长，逐渐形成“可编排工具箱”。

4. PRO 能力差异化
专业版在高级功能、自动化深度和在线扩展上明显更强。

## 使用建议

- 跨版本升级前先备份配置
- 关注版本说明中的行为变化项
- 遇到异常优先回看最近两个版本的变更
