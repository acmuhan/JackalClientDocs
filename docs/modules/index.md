# 模块介绍

JackalClient 采用“按模块启用”的设计：每个能力都是一个独立模块，可单独开关、单独配置，也可以通过快捷键、命令或脚本组合成工作流。


## 模块可用性分级（普通 / PRO）

为了减少“模块很多但不知道能不能用”的困扰，建议先按分级看：

- <Badge type="tip" text="普通可用" />：默认构建通常可用，适合优先尝试。
- <Badge type="warning" text="PRO" />：需要专业版或特定构建支持。
- <Badge type="danger" text="高风险" />：涉及权限提升、系统策略等，先看《安全与边界》。

你可以先从模块地图中的分级速查进入，再跳转到具体模块页面：[/module-map](/module-map)

## 如何阅读模块文档

每个模块页通常包含以下信息：

- **模块用途**：它解决什么问题、适合哪些场景。
- **需求与边界**：是否需要管理员权限、是否依赖驱动、是否依赖联网。
- **配置项说明**：参数含义、默认值与调参建议。
- **关联模块**：与哪些模块可联动，避免功能重叠或冲突。

建议首次使用时，先看“需求”再看“配置项”，最后再做联动配置。

## 使用建议（新手到进阶）

### 1) 新手路径

1. 先阅读《快速开始》，完成基础环境确认与首次运行。[/quick-start](/quick-start)
2. 再阅读《功能与模块地图》，按场景定位你需要的模块类别。[/module-map](/module-map)
3. 从菜单中的“模块文档索引”进入单模块页面，优先使用默认配置。

### 2) 进阶路径

- 结合《完整命令手册》，为常用模块绑定命令或脚本批处理。[/commands/](/commands/)
- 把高频动作拆成“小模块 + 快捷键 + 命令”三段式流程，便于维护和排障。
- 大范围改动前先做配置备份，逐项验证，避免一次性调整过多参数。

## 常见模块场景

- **界面与显示类**：HUD、窗口效果、提示信息可视化。
- **系统与进程类**：进程管理、窗口控制、系统行为调优。
- **自动化类**：按键辅助、流程触发、重复操作自动执行。
- **在线服务类**：B 站相关能力、歌词与在线信息增强。

## 风险与排障建议

- 模块未生效时，优先检查：**权限级别、系统版本、驱动状态、软件冲突**。
- 若出现行为异常，建议只保留单模块复现，再逐步恢复联动配置。
- 对系统限制类模块（如策略、注册表、系统开关）务必在可回滚前提下使用。
